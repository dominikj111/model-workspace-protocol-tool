# Contributing to MWP

The project is in specification stage — no implementation exists yet. The most valuable contributions right now are to the design.

## How to contribute

**Open an issue** for any of the following:

- A gap or inconsistency you found in the spec
- A design question the spec doesn't answer
- A use case the current design doesn't handle
- A community module idea (stack, language idiom, deployment pattern)

**Open a pull request** if you want to propose a concrete spec change. Keep each PR focused on one section or one design question. Reference the relevant section number from `proposal.md`.

## Code contributions

This repository will contain the implementation once the spec stabilises. The roadmap is in [`proposal.md`](proposal.md) — phases are ordered and intentional.

Before opening a code PR, open an issue first to discuss the approach. Early implementation phases especially benefit from a design conversation before any code is written — the spec may need to be refined alongside the code, and that is much easier to do before rather than after. A PR that arrives without prior discussion may be held until the relevant design questions are settled.

## Tone

The spec uses precise language deliberately — "must", "may", "is an error" mean specific things. When proposing changes, match that register and say what you mean exactly.
